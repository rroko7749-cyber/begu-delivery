# Бизнес-логика БегуДоставка

## Все сценарии работы системы

---

## 1. Регистрация и авторизация

### Регистрация клиента

1. Клиент вводит номер телефона
2. Система отправляет SMS с кодом (6 цифр)
3. Клиент вводит код
4. Система проверяет код (3 попытки, 10 мин)
5. Если верно → создаётся аккаунт
6. Клиент вводит имя
7. Email опционально (для чеков)

### Регистрация курьера

1. Курьер вводит номер телефона
2. SMS-код подтверждения
3. Заполняет профиль:
   - Имя, фамилия
   - Тип транспорта
   - Город работы
4. Загружает документы:
   - Фото паспорта
   - Водительские права (если авто/мото)
   - Фото транспорта
5. Вносит депозит 5000₽ (для наличных заказов)
6. Статус: "На проверке"
7. Админ проверяет документы (1-24 часа)
8. Если ОК → статус "Верифицирован", может работать

### Авторизация

1. Ввод номера телефона
2. SMS-код
3. Вход в систему
4. JWT токен (access 15 мин, refresh 30 дней)

## 2. Создание заказа (Клиент)

### Шаг 1: Адреса

1. Клиент нажимает "Создать заказ"
2. Выбирает точки на карте или вводит адреса:
   - Точка A (откуда забрать)
   - Точка B (куда доставить)
   - Опционально: точки C, D, E... (промежуточные)
3. Для каждой точки:
   - Адрес
   - Контактное лицо (имя, телефон)
   - Тип (забрать/доставить)
   - Примечания

### Шаг 2: Детали груза

1. Описание груза (текст)
2. Вес (выбор: до 5кг, 5-10кг, 10-20кг, 20-30кг, >30кг)
3. Фото груза (опционально)

### Шаг 3: Срочность

- Обычная (60 мин) - базовая цена
- Быстрая (30 мин) - +40%
- Экспресс (15 мин) - +100%

### Шаг 4: Расчёт цены

Система автоматически рассчитывает:
```
Базовая цена: 240₽
  - Расстояние (5 км): 240₽
  - Доп. точка: +80₽
  
Коэффициенты:
  - Час пик (18:00): ×2.0
  - Спрос высокий: ×1.5
  - Дождь: ×1.3
  
Итого: 240₽ × 2.0 × 1.5 × 1.3 = 936₽
```

Клиент видит разбивку и может:
- Принять цену
- Отменить

### Шаг 5: Способ оплаты

- Наличные курьеру
- Карта (онлайн)
- Баланс в приложении

### Шаг 6: Подтверждение

1. Клиент подтверждает заказ
2. Если оплата картой → списание сразу
3. Заказ создан, статус: "Ищем курьера"

---

## 3. Поиск курьера

### Автоматический поиск

1. Система находит всех курьеров:
   - В радиусе 5 км от точки A
   - Онлайн и не заняты
   - Подходящий транспорт
   - Рейтинг > 4.0

2. Сортирует по:
   - Расстояние до точки A
   - Рейтинг
   - Количество выполненных заказов

3. Отправляет push-уведомление топ-10 курьерам:
   ```
   Новый заказ рядом!
   5 км, 2 точки, 936₽
   Ваш заработок: 796₽
   ```

4. Курьеры видят:
   - Точку A на карте (без адреса)
   - Расстояние
   - Количество точек
   - Цену
   - Примерное время

5. Первый принявший → получает заказ

### Если никто не принял (5 минут)

1. Система предлагает клиенту:
   ```
   Курьер не найден
   Хотите повысить цену?
   
   [+50₽] [+100₽] [+200₽] [Своя сумма]
   ```

2. Клиент повышает цену
3. Доплата идёт полностью курьеру
4. Уведомление курьерам снова

### Если снова никто не принял

1. Расширяем радиус до 10 км
2. Снижаем требования к рейтингу (> 3.5)
3. Ещё 5 минут ожидания

### Если всё равно никто

1. Заказ отменяется автоматически
2. Деньги возвращаются клиенту
3. Извинения + промокод на скидку

---

## 4. Выполнение заказа (Курьер)

### Принятие заказа

1. Курьер видит уведомление
2. Нажимает "Принять"
3. Система проверяет:
   - Курьер всё ещё онлайн?
   - Не занят другим заказом?
   - Заказ ещё не принят другим?
4. Если ОК → заказ назначен
5. Курьер видит полную информацию:
   - Все адреса
   - Контакты
   - Описание груза
   - Маршрут на карте

### Движение к точке A

1. Статус: "Курьер едет к отправителю"
2. Клиент видит курьера на карте (обновление каждые 5 сек)
3. Навигация для курьера (Яндекс.Карты)
4. ETA (примерное время прибытия)

### Прибытие к точке A

1. Курьер нажимает "Я на месте"
2. Статус: "Курьер прибыл"
3. Клиент получает уведомление
4. Курьер звонит/пишет отправителю

### Получение груза

1. Курьер получает груз
2. Может сфотографировать (для подтверждения)
3. Нажимает "Груз получен"
4. Статус: "Груз в пути"
5. Код подтверждения (6 цифр) - отправитель диктует курьеру
6. Курьер вводит код → подтверждение

### Движение к точке B (или C, D...)

1. Навигация к следующей точке
2. Клиент видит курьера на карте
3. ETA обновляется

### Прибытие к точке B

1. Курьер нажимает "Я на месте"
2. Уведомление получателю
3. Курьер звонит/пишет

### Доставка груза

1. Курьер передаёт груз
2. Фото доставки (опционально)
3. Получатель диктует код подтверждения
4. Курьер вводит код
5. Нажимает "Доставлено"

### Если есть ещё точки

Повторяем для точек C, D, E...

### Завершение заказа

1. Все точки пройдены
2. Статус: "Доставлено"
3. Если оплата наличными:
   - Курьер получает деньги от клиента
   - Комиссия списывается с депозита курьера
4. Если оплата картой:
   - Деньги уже списаны
   - Курьеру начисляется заработок на баланс
5. Статус: "Завершён"

## 5. Оплата и выплаты

### Оплата клиентом

**Наличные:**
- Клиент платит курьеру при получении
- Курьер подтверждает получение в приложении
- Комиссия 15% списывается с депозита курьера

**Карта:**
- Списание при создании заказа
- Холд на сумму заказа
- После завершения → списание окончательное
- Если отмена → возврат

**Баланс:**
- Клиент пополняет баланс заранее
- При заказе списывается с баланса
- Бонусы и промокоды

### Выплаты курьеру

**Баланс курьера:**
```
Начальный депозит: 5000₽
Заработок за заказ: +796₽
Комиссия (наличный заказ): -140₽
Текущий баланс: 5656₽
```

**Вывод средств:**
- Минимум: 500₽
- На карту (мгновенно, комиссия 0%)
- На счёт (1-3 дня, комиссия 0%)
- Лимит: 3 вывода в день

**Автовывод:**
- Настройка: "Выводить автоматически при достижении 5000₽"
- Деньги приходят на привязанную карту

## 6. Отмены и споры

### Отмена клиентом

**До принятия курьером:**
- Отмена бесплатно
- Возврат 100%

**После принятия, но курьер не выехал:**
- Штраф 50₽
- Возврат 95%

**Курьер уже едет:**
- Штраф 100₽ + компенсация курьеру
- Возврат 80%

**Груз уже получен:**
- Отмена невозможна
- Нужно создать новый заказ обратно

### Отмена курьером

**До получения груза:**
- Штраф курьеру: -100₽ с депозита
- Рейтинг: -0.1
- Заказ возвращается в поиск

**После получения груза:**
- Отмена невозможна
- Нужно связаться с поддержкой

### Споры

**Клиент жалуется:**
- "Курьер не приехал"
- "Груз повреждён"
- "Курьер грубил"

**Курьер жалуется:**
- "Клиент не вышел"
- "Неверный адрес"
- "Клиент отказался платить"

**Процесс:**
1. Обращение в поддержку
2. Создаётся тикет
3. Поддержка изучает:
   - GPS-трек курьера
   - Фото
   - Переписку
   - Коды подтверждения
4. Решение в течение 24 часов
5. Возможные исходы:
   - Возврат денег клиенту
   - Компенсация курьеру
   - Блокировка нарушителя
   - Отклонение жалобы

## 7. Рейтинги и отзывы

### После завершения заказа

1. Клиент оценивает курьера (1-5 звёзд)
2. Курьер оценивает клиента (1-5 звёзд)
3. Опционально: текстовый отзыв

### Влияние рейтинга

**Курьер:**
- < 4.0 → меньше заказов
- < 3.5 → предупреждение
- < 3.0 → блокировка

**Клиент:**
- < 4.0 → курьеры видят предупреждение
- < 3.0 → сложнее найти курьера

---

## 8. Промокоды и бонусы

### Промокоды

**Типы:**
- Скидка % (например, -20%)
- Скидка фикс (например, -100₽)
- Бесплатная доставка

**Применение:**
1. Клиент вводит промокод при заказе
2. Система проверяет:
   - Код существует?
   - Не истёк?
   - Не исчерпан лимит?
   - Клиент не использовал ранее?
3. Если ОК → скидка применяется
4. Цена пересчитывается

### Реферальная программа

**Клиент приглашает друга:**
- Друг регистрируется по ссылке
- Делает первый заказ
- Оба получают бонус 200₽ на баланс

**Курьер приглашает курьера:**
- Новый курьер регистрируется
- Выполняет 10 заказов
- Оба получают бонус 1000₽

## 9. Уведомления

### Push-уведомления

**Клиент:**
- Курьер принял заказ
- Курьер прибыл
- Груз получен
- Груз доставлен
- Заказ завершён
- Ответ от поддержки

**Курьер:**
- Новый заказ рядом
- Заказ отменён
- Новое сообщение от клиента
- Пополнение баланса
- Вывод средств выполнен

### SMS

- Код подтверждения при входе
- Критичные уведомления (блокировка, споры)

### Email

- Чек после заказа (если указан email)
- Еженедельная статистика
- Промо-акции

## 10. Безопасность

### Предотвращение мошенничества

**Фейковые заказы:**
- Лимит 3 отмены в день
- После 5 отмен → блокировка на 24 часа
- Проверка номера телефона

**Фейковые курьеры:**
- Верификация документов
- Депозит 5000₽
- Первые 10 заказов → на контроле

**Накрутка рейтинга:**
- Алгоритм детекции подозрительных отзывов
- Нельзя оценить одного курьера > 3 раз в месяц

### Защита данных

- Все пароли: bcrypt
- Персональные данные: шифрование AES-256
- HTTPS везде
- JWT токены с коротким TTL
- Логирование всех действий
