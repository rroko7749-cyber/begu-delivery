## 🚀 Что можно тестировать

### 1. Регистрация курьера
1. Регистрация через SMS
2. Заполнение профиля курьера
3. Загрузка документов (паспорт, права, фото транспорта)
4. Оплата депозита 5000₽ через ЮKassa
5. Верификация админом
6. Начисление депозита на баланс

### 2. Создание заказа
1. Клиент создаёт заказ с несколькими точками
2. Система рассчитывает цену (240₽ база + коэффициенты)
3. Автоматический поиск курьеров в радиусе 5км
4. Уведомления топ-10 курьерам
5. Если не найдено - расширение до 10км
6. Таймер 5 минут - предложение повысить цену
7. Таймер 10 минут - автоотмена с возвратом

### 3. Выполнение заказа
1. Курьер принимает заказ
2. Курьер прибывает на точку отправления
3. Курьер вводит код подтверждения от отправителя
4. Курьер забирает груз
5. Курьер доставляет на точку назначения
6. Курьер вводит код подтверждения от получателя
7. Заказ завершается автоматически
8. Курьеру начисляется 85% от суммы

### 4. Платежи
1. Оплата заказа картой через ЮKassa
2. Оплата депозита курьера
3. Обработка наличных (списание комиссии)
4. Вывод заработка курьером
5. Возврат средств при отмене

## ⚙️ Настройка для тестирования

### 1. Переменные окружения (.env)

```env
# Database
DB_HOST=localhost
DB_PORT=5432
DB_NAME=begudelivery
DB_USER=postgres
DB_PASSWORD=your_password

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# JWT
JWT_SECRET=your_jwt_secret_key
JWT_ACCESS_EXPIRY=15m
JWT_REFRESH_EXPIRY=30d

# SMS Provider
SMS_API_KEY=your_sms_api_key

# YooKassa (тестовые ключи)
YOOKASSA_SHOP_ID=your_test_shop_id
YOOKASSA_SECRET_KEY=your_test_secret_key

# Frontend URL
FRONTEND_URL=http://localhost:19006

# S3 Storage
S3_ENDPOINT=https://s3.amazonaws.com
S3_REGION=us-east-1
S3_ACCESS_KEY=your_access_key
S3_SECRET_KEY=your_secret_key
S3_BUCKET=begudelivery

# Weather API
WEATHER_API_KEY=your_weather_api_key

# Commission
COMMISSION_RATE=0.15
```

### 2. Запуск

```bash
# Миграция БД
npm run migrate

# Запуск сервера
npm run dev
```

### 3. Webhook ЮKassa

Для локального тестирования webhook используйте ngrok:

```bash
ngrok http 3000
```

Затем в личном кабинете ЮKassa укажите:
```
https://your-ngrok-url.ngrok.io/api/v1/payments/webhook/yookassa
```

## 🐛 Известные ограничения

1. **Push-уведомления** - пока только в БД, нет интеграции с FCM
2. **Выплаты курьерам** - упрощённая схема, нужен договор с ЮKassa для Payouts API
3. **Коэффициент спроса** - hardcoded 1.0, нужна динамическая логика
4. **WebSocket** - нет real-time трекинга курьера
5. **Чаты** - нет общения между клиентом и курьером
